实现苏宁移动端首页(静态页面)

发表于 2021-06-30 更新于 2021-07-13 分类于 Web 阅读次数：

2021.6.30

为了更好的掌握各种布局和样式，模仿着写了几个移动端的首页。
分别采用了不同的布局方式。这个例子主要采用rem+less的适配方式。
有时间的话会继续优化。

需要注意的地方有：
1.合理用less。
2.知道通过rem怎么适配不同机型的原理。
3.了解em和rem的区别。
4.了解flexible.js 适配不同机型的原理。
5.了解一些插件的使用，如easyless和cssrem。
6.与前面两种布局的区别。

效果如图

太多的东西和之前的两个页面一样。所以这个页面只实现了部分。
只要是学会如何用rem适配不同设备。

rem是根据html中font-size来设置大小的，所以只要根据不同机型设置不同的font-size，这样的比例相同，相同的rem值在不同的设备中因为font-size的不同会表示不同的大小，从而达到适配的效果。

因为可能会涉及到计算，没怎么用过less，这里采用less练练手。
先写好适配不同机型的common.less

通过eazyless插件会把这个文件生成相应的css文件。

在为页面创建相应的less文件，把这个文件引入之后，在写入自己页面的东西即可。

前面的less文件中用到了很多除法实际上，可以使用cssrem插件来让插件自动帮我计算。

另外也可以使用flexible来帮我们完成不同设备的适配，其实主要原理和我们手动划分份数计算比例差不到。只不过我划分成了15份，而他划分成了10份。实际上不会有太多区别。

最后使用这种方式和百分比布局以及flex布局的区别，我个人认为用了前两种的布局的大多数情况下，都是高度固定了来通过改变宽度的比例来适配不同的页面，这样导致的问题就是，无论是放大还是缩小都仅仅只是宽度的等比例缩放达到适配，如果真正要响应式布局的话还是需要宽度高度都有一个明确的可以根据不同设备变化的值，rem无疑就是我们等的那个人。
